    【Spring源码学习】0x07 BeanDefinition加载之spring-beans.xsd文件

    vuePress-theme-reco iYoung    2021

    【Spring源码学习】0x07 BeanDefinition加载之spring-beans.xsd文件

    iYoung 2021-5-11 Spring

    # 场景

    上篇介绍了Spring XML配置文件的校验是怎么做的，提到了XSD文件，那这个就来看看这个spring-beans.xsd文件到底定义了什么？我在配置Spring的时候，都能配置哪些东西？

    # 分析

    XSD还是比较简单的，它本身就是一个XML文件，不过XSD是一种有约束的配置文件，也就是说它的格式是规定好的，有哪些节点，节点可以有哪些属性，都规范好了。

    以固定格式的XSD，规范无固定格式的XML，又不牺牲其灵活性，不得不说，这种思路，很神奇。

    直接看spring-beans.xsd，这个文件的路径是spring-beans/org/springframework/beans/factory/xml/spring-beans.xsd。

    打开这个文件，不得不说，注释是真多，所以这个文件就是给开发者看的，可惜，我以前从来没有看过。

    先看上面的注释中的标题：Spring XML Beans Schema, version 4.3，我Spring版本使用的是5.3.6，但是这个Schema的版本还是4.3，看来5.x以来，对于BeanDefinition应该是没有什么改动的。

    这里记录一些题外话，为什么我要从基本过时的XML开始学习，因为这是Spring的最开始的样子。 从这里开始学习，涉及到的概念可能会少一些，精力可以更专注一些，抛出分支剧情，直指核心问题，然后再逐步增加新特性，一步一步扩展开来。

    # 概述

    这开头的一段话，信息量还是蛮大的，简单总结一下。

    开篇明义，这个文件就是定义Spring容器管理的Bean对象的，还交代了这个文件是XmlBeanDefinitionReader中DefaultBeanDefinitionDocumentReader使用的。

    这里直接交代了bean节点的本质：每个bean节点都定义了一个JavaBean。

    JavaBean

    JavaBean是一个规范 (opens new window)，是有一定要求的：

    1. 可序列化，即实现了java.io.Serializable
    2. 无参构造器
    3. 私有属性配合getter & setter

    JavaBean其目的就是封装一系列的数据，提供公开的读写方式，达到组件重用的目的。 说白了，就是一种数据结构，或者说是结构数据。

    这个JavaBean在这里没有明说，但其实就是BeanDefinition，那这里也就明确了BeanDefinition的角色和功能。

    然后，文档又讲到:bean实例分为单例和原型，但是可以通过扩展提供更多的Scope。

    而且，bean是可以互相引用的，可以作为其他bean的属性或者参数，但是仅限在同一BeanFactory或者祖先BeanFactory中。

    除了引用，还可以建立内部bean，仅限包裹它的bean作用域内使用。

    同样支持list、set、map、properties类型。

    这个概述全是干货。

    下面就是定义文档结构的。

    # beans

    我一直觉得，而且也一直是这么用的：beans属于根结点。 其实这种表述不是那么正确，文档是这么描述的：beans是<bean>和其他一些元素的容器，通常作为文档根结点。

    另外，之前一直没有注意和使用过，beans节点不仅是作为根结点使用的，还可以设置默认属性，供内部定义的bean使用。而且beans是可以嵌套beans的，可以对部分bean设置默认值，也可以通过profile的形式，启用或关闭某些bean。

    XSD不仅定义文档节点和属性，同样可以定义节点出现的顺序和次数。

    # 子节点

    # description

    描述信息，可选的，至多出现一次。 如果出现了该节点，则必须放在第一个。

    <beans>
    <!-- 这里不会有其他节点 -->
    <description>这个配置文件用于拯救地球</description>
    <!-- 从这开始定义其他节点 -->
</beans>
    # beans

    上面说的嵌套的beans节点，也是可选的，至多N次。 如果出现该节点，该节点要放在文档的最后。

    <beans>
    <!-- 这里可以定义其他节点 -->
    <beans>
        <!-- 一堆配置 -->
    </beans>
    <!-- 这里不能再定义其他节点了 -->
    <beans>
        <!-- 一堆配置 -->
    </beans>
    <!-- 这里不能再定义其他节点了 -->
</beans>
    # import

    导入一个XML配置资源，可选，至多N次。

    当Spring XML配置文件过大，元素过多时，将其分割成多个配置文件便于维护，然后使用import的方式，将其汇总。

    import节点有个resource属性，用于指定资源路径。

    <beans>
    <import resource="other.xml"/>
    <import resource="../other.xml"/>
    <import resource="classpath:other.xml"/>
    <import resource="classpath*:spring*.xml"/>
</beans>

    这个支持多种格式，能支持多少种，之后看代码实现吧。

    # alias

    定义别名，可选，至多N次。

    这个可以被bean定义别名，而且是可以跨配置文件的，上面import那种情况也是可以的。

    alias有两个属性：name和alias，name就是指定bean的，alias是用来设置别名的。

    <beans>
    <bean id="idxxx" name="xxx" class="..."></bean>
    <alias name="xxx" alias="x3">
    <alias name="idxxx" alias="idx3">
</beans>
    # bean

    这个bean节点内容就比较多了，下面单独看吧。

    # 属性

    这里看看，beans都支持哪些属性。

    # profile

    指定配置场景，支持配置多个，以空格/逗号/分号分割，可选。

    通过指定配置场景，可以按需注册需要的bean或者配置。

    <beans profile="dev,test;pro">
</beans>
    # default-lazy-init

    延迟初始化的默认配置，对应bean中的lazy-init属性，默认值是default。

    提示

    deafult代表继承外部beans的这个属性，一直往上找，如果没有找到就按照false处理。

    # default-merge

    这个属性是针对merge属性的，默认值为default。

    这个merge属性是针对集合/Map的，在bean的继承情况下，如果某个属性属于集合/Map，那就存在两种处理方式：merge和override。

    merge为true时，父子bean的值就会merge在一起；merge为false时，子bean就是覆盖父bean的值。

    # default-autowire

    对应autowire属性，默认值为default。 设置自动装配的方式。

    # default-autowire-candidates

    对应autowire-candidate属性，在按类型自动装配时，确定bean是否作为注入的候选bean。 但是与autowire-candidate有些不同，这里是配置通配符："*Service", "data*", "*Service*", "data*Service"。

    # default-init-method

    对应init-method方法，默认值为default。

    # default-destroy-method

    对应destroy-destroy方法，默认值为default。

    # bean

    bean节点，定义一个bean，对应BeanDefinition。这个有很多属性，也可以嵌套很多节点。

    # 子节点

    # description

    bean节点也有description子节点，同样的，也是只能放在第一个，至多出现一次。

    <bean>
    <!-- 这里不能定义其他节点 -->
    <description>我是描述</description>
    <!-- 这里可以定义其他节点 -->
</bean>
    # meta

    绑定在bean上的元数据，一个键值对，可以任意多个，有两个属性：key和value。

    这个与property节点不同之处在于，property对应的是bean类型内部的字段，而meta仅在bean定义中使用。

    <bean>
    <meta key="foo" value="bar" />
</bean>
    # constructor-arg

    注入构造器参数，配置了这个，在初始化bean的时候，就会把这些参数解析出来，注入进去。

    这个也可以配置一个description的子节点。

    其属性有：

    • index
    • name
    • type
    • ref
    • value

    从这些属性来看，支持索引，参数名，参数类型方式，参数值支持bean引用和值的方式。

    另外，参数值还可以采用嵌套子节点的方式：

    • bean
    • ref
    • idref
    • value
    • null
    • array
    • list
    • set
    • map
    • props
    # property

    这个就是bean类型的字段，符合JavaBean命名。

    # qualifier

    这个注入时的限定条件，可以为注入更好查找候选bean。

    # lookup-method

    配置该节点，会让Spring容器覆盖指定方法的实现，返回指定的bean，该方法会触发getBean方法。 这个具体用法之后再看吧。

    # replaced-method

    该节点的作用是方法注入，替换实现。

    # 属性

    • id
    • name
    • class
    • parent
    • scope
    • abstract
    • lazy-init
    • autowire
    • depends-on
    • autowire-candidate
    • primary
    • init-method
    • destroy-method
    • factory-method
    • factory-bean

    这些属性前面都看到过，具体使用场景在后面的XML解析中再看吧。

    # 结论

    这个XSD就是BeanDefinition的说明文档，属性和节点有很多，具体的使用场景还是要到代码里去看。
