    【Spring源码学习】0x06 BeanDefinition加载之XML校验

    iYoung 2021-5-9 Spring

    前面容器初始化过程已经分析结束，大致流程清楚了，但是屏蔽了太多细节，现在就看看BeanDefinition加载的过程中，XML是如何校验的。

    # 场景

    回头看配置文件：

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
       http://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean class="xyz.iyoung.study.spring.xml.GreetingService" />

</beans>

    我稍微修改了一下，<bean test="dummy" class="xyz.iyoung.study.spring.xml.GreetingService" />， 然后重新执行代码，就报错了：元素 'bean' 中不允许出现属性 'test'。

    为什么报错呢？

    # 分析

    对于XML来说，它仅仅是纯文本，结构上是松散的，没有预定义标签或属性，所以从XML的角度来说，下面的XML是合法的：

    <beans>
    <bean class="xyz.iyoung.study.spring.xml.GreetingService" />
    <bean test="dummy" class="xyz.iyoung.study.spring.xml.GreetingService" />
    <bean test="dummy" class="xyz.iyoung.study.spring.xml.GreetingService" />
</beans>

    如果，这个XML交给Summer来处理，Summer觉得合适，那这个XML就是有效的； 如果Winter能识别，那对于Winter来说，也可以。 但是，这个XML交给Spring来处理，Spring说，我不认可。

    所以，对于过于松散的XML结构，定义了另一种结构，来规范它，这就是XML Schema。

    XML Schema在这里就先不展开，以后再说。

    先看看，代码中是怎么验证的。

    前面已经知道，XML的读取，解析是在XmlBeanDefinitionReader中真正开始的：

    protected Document doLoadDocument(InputSource inputSource, Resource resource) throws Exception {
    return this.documentLoader.loadDocument(inputSource, getEntityResolver(), this.errorHandler,
            getValidationModeForResource(resource), isNamespaceAware());
}

    这里委托给了DefaultDocumentLoader进行处理，先看方法声明：

    Document loadDocument(
			InputSource inputSource, EntityResolver entityResolver,
			ErrorHandler errorHandler, int validationMode, boolean namespaceAware)
			throws Exception;

    看看这些参数：

    1. inputSource 这个就是XML文件流。
    2. entityResolver 在AbstractXmlApplicationContext#loadBeanDefinitions(DefaultListableBeanFactory)这个方法中，创建了XmlBeanDefinitionReader对象，顺便设置了entityResolver：
      beanDefinitionReader.setEntityResolver(new ResourceEntityResolver(this));
    3. errorHandler
      private ErrorHandler errorHandler = new SimpleSaxErrorHandler(logger);
    4. validationMode 校验的模式：DTD或者XSD，这里使用对的是XSD。
    5. namespaceAware 是否识别XML的命名空间，这里是默认值false。

    通过这些参数，就可以直达奥DefaultDocumentLoader#loadDocument这个方法解析XML文件的时候使用了XSD模式，对XML进行了校验。

    但还有个问题，那XSD文件呢？

    我们看看这个ResourceEntityResolver，继承自DelegatingEntityResolver，从名字上来看，又是这个DelegatingEntityResolver在做事：

    public InputSource resolveEntity(@Nullable String publicId, @Nullable String systemId)
			throws SAXException, IOException {
    if (systemId != null) {
        if (systemId.endsWith(DTD_SUFFIX)) {
            return this.dtdResolver.resolveEntity(publicId, systemId);
        }
        else if (systemId.endsWith(XSD_SUFFIX)) {
            return this.schemaResolver.resolveEntity(publicId, systemId);
        }
    }

    // Fall back to the parser's default behavior.
    return null;
}

    systemId就是XML配置文件中的http://www.springframework.org/schema/beans/spring-beans.xsd，那既然已经拿到地址了，我直接就去下载不就好了，但Spring却没有这么做，而是交给了PluggableSchemaResolver#resolveEntity去做：

    public InputSource resolveEntity(@Nullable String publicId, @Nullable String systemId) throws IOException {
    if (systemId != null) {
        String resourceLocation = getSchemaMappings().get(systemId);
        if (resourceLocation == null && systemId.startsWith("https:")) {
            // Retrieve canonical http schema mapping even for https declaration
            resourceLocation = getSchemaMappings().get("http:" + systemId.substring(6));
        }
        if (resourceLocation != null) {
            Resource resource = new ClassPathResource(resourceLocation, this.classLoader);
            try {
                InputSource source = new InputSource(resource.getInputStream());
                source.setPublicId(publicId);
                source.setSystemId(systemId);
                return source;
            }
            catch (FileNotFoundException ex) {
                if (logger.isDebugEnabled()) {
                    logger.debug("Could not find XML schema [" + systemId + "]: " + resource, ex);
                }
            }
        }
    }

    // Fall back to the parser's default behavior.
    return null;
}

    代码看着比较多，实际上有用的就那么两行：

    String resourceLocation = getSchemaMappings().get(systemId);
Resource resource = new ClassPathResource(resourceLocation, this.classLoader);

    看看这个getSchemaMappings方法：

    private Map<String, String> getSchemaMappings() {
    // ...
    Properties mappings = PropertiesLoaderUtils.loadAllProperties(this.schemaMappingsLocation, this.classLoader);
    // ...
}

    这个文件的格式就是systemId=路径，在Spring的jar包里面存放了schema文件，Spring没有去网络上请求，而是使用的本地文件。

    就这样，XSD文件也找到了。

    # 结论

    总结一下这个流程：

    1. 加载XML文件流
    2. 创建解析XML工厂
    3. 加载本地Schema文件
    4. 解析校验XML文件

    所以，在校验阶段，代码报错。
